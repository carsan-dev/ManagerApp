Para ejecutar el proyecto en android:

- npm start -- --reset-cache
- npm run android

Para matar el proceso si se queda pillado:

- npx kill-port 8081

Para compilar en .exe:

- cd android/
- ./gradlew clean
- cd ..
- npx react-native bundle --platform android --dev false --entry-file index.js --bundle-output android/app/src/main/assets/index.android.bundle --assets-dest android/app/src/main/res
- cd android/
- ./gradlew assembleDebug

 Procedimiento para cambiar de versión:

  1. Actualizar version en package.json (ej: 3.1.0)
  2. Actualizar versionCode y versionName en android/app/build.gradle
  3. Generar el APK
  4. Subir el APK a tu servidor/hosting
  5. Actualizar latest_version y update_url en Firebase Remote Config
  6. Si es crítica, pon force_update: true

Usar google drive para alojar el apk y subirlo a firebase:

  1. Subir el APK a Drive y compartirlo (cualquiera con el enlace)
  2. Copiar el link
  3. Extraer el ID (la parte entre /d/ y /view):
  1aBcDeFgHiJkLmNoPqRsTuVwXyZ
  4. Crear el link de descarga directa:
  https://drive.google.com/uc?export=download&id=1aBcDeFgHiJkLmNoPqRsTuVwXyZ
  5. En Firebase Remote Config, poner ese link en update_url
